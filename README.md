## Prerequisites

- Java JDK 17 or later
- MySQL Server 8.0+
- MySQL Connector/J (`mysql-connector-java.jar`) — already in `lib/`
- FlatLaf (`flatlaf-3.4.1.jar`) — already in `lib/`

## Setup Database

1. Open MySQL command line or any MySQL client (e.g., MySQL Workbench, phpMyAdmin)
2. Run the schema script:
   ```
   source sql/schema.sql
   ```
    Or copy and paste the contents of `sql/schema.sql` into your MySQL client.

## Configure Database Connection

Edit `src/school/enrollment/database/DatabaseConnection.java` to match your MySQL credentials:

```java
private static final String URL = "jdbc:mysql://localhost:3306/school_enrollment";
private static final String USER = "root";
private static final String PASSWORD = "";  // change if needed
```

## How to Run

### Option 1 — Using the batch scripts (Windows):

```cmd
build       # compiles all sources into build/
run         # runs the application
```

### Option 2 — Command Prompt (cmd.exe):

```cmd
javac -d build -cp "lib\*" @sources.txt
java -cp "build;lib\*" school.enrollment.Main
```

### Option 3 — PowerShell:

```powershell
javac -d build -cp "lib\*" src\school\enrollment\*.java src\school\enrollment\model\*.java src\school\enrollment\dao\*.java src\school\enrollment\daoimpl\*.java src\school\enrollment\database\*.java src\school\enrollment\controller\*.java src\school\enrollment\view\*.java src\school\enrollment\theme\*.java
```

## Features

- **Student Registration** – Add, update, and search students with custom ID format (XXXX-XXXX). Uses **soft-delete (Active/Inactive)** — deactivated students are hidden from lists but their records are preserved.
- **Course Management** – Add, update, and search courses with computed total tuition. Courses with active enrollments **cannot be deactivated**. Uses **soft-delete (Active/Inactive)**.
- **Enrollment** – Enroll students in **multiple courses at once** with a single combined success dialog. Enrollment history is tracked in the audit log.
- **Drop Enrollment** – Instead of permanent deletion, enrollments are marked as **"Dropped"**. Drop is **blocked if payment has already been made** (admin refund required).
- **Tuition Payment** – Select subjects via checkbox, pay per subject or all at once, supports Cash, Bank Transfer, and Check.
- **Architecture** – MVC + DAO design pattern, JDBC for database connectivity, Swing (JFrame) for GUI with FlatLaf theme.
- **Computation** – Total tuition = units × tuition_per_unit, Balance = total − paid.

## Architecture

- **MVC** (Model-View-Controller)
- **DAO** (Data Access Object) design pattern
- **JDBC** for database connectivity
- **Swing (JFrame)** for GUI
- **FlatLaf** (FlatLightLaf) for modern look and feel
